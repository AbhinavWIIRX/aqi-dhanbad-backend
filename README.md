## Critical Data Issue Found

Your `AQI` column contains values **1–5** (AQI category/bucket), NOT the real 0–500 AQI index.

The training code automatically computes the real AQI from PM2.5, PM10, NO2, SO2, CO, O3 using the CPCB sub-index formula. You do NOT need to fix your CSV.

Also: 1 row has PM10 = -9999 (sensor dropout) — automatically dropped during training.

## Training

### Quick train (no tuning, ~3 min)
```bash
cd src
python train.py --mode both
```

### With Optuna hyperparameter tuning (~30 min, better results)
```bash
python train.py --mode both --tune
```

### Train only one model
```bash
python train.py --mode single    # just AQI direct
python train.py --mode dual      # just PM2.5 + PM10
```

Expected results on test set (last 15% of data, time-ordered):
- Single AQI model: RMSE ~25–40 AQI units, R² ~0.85–0.92
- Dual PM2.5: RMSE ~12–18 µg/m³, R² ~0.88–0.93
- Dual PM10:  RMSE ~15–25 µg/m³, R² ~0.86–0.91

## Feature Engineering Details

### Features created from your dataset + OWM

| Feature group | Features |
|---|---|
| Weather (from OWM) | Temperature, Humidity, WindSpeed |
| Pollutants (from OWM Air API) | CO, NO, NO2, O3, SO2, NH3 |
| Cyclical time | hour_sin/cos, month_sin/cos, dow_sin/cos |
| Calendar | is_weekend, is_winter, is_monsoon, is_peak_hour, is_festival |
| Lag features | AQI/PM2.5/PM10 at t-1h, t-3h, t-6h, t-12h, t-24h, t-48h |
| Rolling stats | 6h/12h/24h rolling mean + std |
| Wind direction | wind_sin, wind_cos (encoded from degrees) |

### Target

**Single model**: `AQI_computed` — computed from PM2.5+PM10+NO2+SO2+CO+O3 via CPCB sub-index formula (max of all sub-indices). Range 0–500.

**Dual model**: `PM2_5` (µg/m³) and `PM10` (µg/m³) separately.

Both use **log1p transform** on target (Dhanbad has extreme right skew — PM2.5 max=688 µg/m³).

---

## Model Architecture

```
Single model
  Input: 50+ features (weather + pollutants + lags + time)
  XGBoost Regressor (reg:squarederror)
  Target: log1p(AQI_computed)
  Output: expm1(prediction) → AQI (0–500)

Dual model
  Input: same 50+ features (with PM2.5/PM10 lags instead of AQI lags)
  XGB₁ → log1p(PM2_5) → expm1 → PM2.5 (µg/m³)
  XGB₂ → log1p(PM10)  → expm1 → PM10 (µg/m³)
  CPCB sub-index formula → AQI (0–500)
```

## Dhanbad-Specific Notes

- **Coal belt effect**: Dhanbad (Jharia coalfield) has chronic PM2.5 from open-cast mining and coal fires. Model trained on this data captures the baseline.
- **Winter haze (Nov–Feb)**: Temperature inversion traps pollutants. Model has `is_winter` flag and winter-month lag patterns.
- **Chhath Puja / Diwali (Oct–Nov)**: Fireworks spike AQI by 200–300 points. `is_festival` flag handles known dates.
- **Monsoon washout (Jun–Sep)**: Rain scavenges particulates. `is_monsoon` + `Rain3h` features from OWM forecast.
